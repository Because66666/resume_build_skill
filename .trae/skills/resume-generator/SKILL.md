# Resume Generator (简历生成器)

根据用户提供的个人经历和岗位需求，自动生成适配的 HTML 简历文件。

## 使用流程

### 1. 收集输入信息

向用户收集以下信息：

- **个人信息**：一个md文件，里面包含用户的个人信息与个人经历，无需格式。
- **岗位信息**：投递的岗位详细描述
- **照片**：提醒用户提供个人照片，照片文件名需AI针对性替换 `template.html` 中的 `<img src="./default.png">` 中的路径

### 1.1 解析个人信息

对于用户提供的个人经历，需要检查其每一段经历是否包含以下要素：
- 经历的时间段或者时间点，例如：2023年1月-2023年6月
- 经历的描述中是否有可以量化的成果，例如：20%的效率提升，服务了100余名同学等。
如果没有，则引导用户进行信息的补充。

### 2. 分析岗位需求

分析岗位 JD，明确岗位侧重于需要什么技能：
- 从"工作职责"中提取核心任务关键词
- 从"任职要求"中提取硬性条件和加分项
- 判断岗位类型（算法、产品、开发、运营等）
- 确定简历中需要重点突出的能力和经验方向

### 3. 筛选个人经历

**核心要求：筛选出最接近岗位需求的三条经历，其余省略。**

筛选标准：
- 匹配度高的经历优先（与岗位职责关键字重叠多）
- 近期的经历优先
- 有量化成果的经历优先
- 筛选完成后，**必须先请示用户进行审核**，用户确认后再继续

### 4. 生成简历 HTML

参考 `template.html` 模板生成简历文件。

模板说明：
- **网页标题**改为：`姓名-来自（大学名称）-（岗位名称）`
- **照片提醒**：询问用户是否提供照片，若有则替换 `<img src="./default.png">` 中的路径，如果没有则需要提醒用户自行替换
- **教育背景**：
  - 学校名称、专业、入学-毕业时间按实际填写
  - 主修课程：筛选与岗位相关的课程，按相关性从高往低排列，前3个用 `<strong>` 标签
- **项目经历**：筛选后的经历，每段需包含：
  - 项目名称（或经历名称）
  - **具体时间范围（具体到年月）**，如果不清楚则写明 `【待补充】`，并在之后的返回中提醒用户补充时间范围
  - 工作内容用一段话描述（使用 `<p>` 标签，不用多个 `<p>`）
- **校园经历**：学生干部、助教等经历，同样写清楚时间范围
- **荣誉奖项**：用 2 行写完，不要分点
- **自我评价**：用 2 行写完
- **专业技能**：分技能和语言两行

### 6. 文件保存

- **HTML 文件**保存在 `/htmls` 目录下，文件名为 `岗位-企业.html`
  - 例如：`多模态大模型算法工程师-字节.html`
- **PDF 相关**：在 `/pdfs` 文件夹下新增子文件夹，文件夹名为 `岗位-企业`
  - 例如：`/pdfs/多模态大模型算法工程师-字节/`
  - 提示用户可以自行在浏览器中打印为 PDF 保存到该目录。步骤为：浏览器内鼠标右键，打印，选择另存为PDF。

### 7. 输出要求

- 每一段经历必须写清楚具体的时间范围（具体到年月）
- 如果不清楚时间，写明 `【待补充】`
- 简历内容要针对岗位需求做适配，突出相关技能和经验
- 结束后通知用户审查，并且要求用户仔细检查简历内容有无编造或错误。建议用户简历另存为pdf后，其页数不超过一页。

## 注意事项

1. 不要自行编造用户没有提到的经历或信息
2. 筛选经历后必须请示用户审核，不能直接跳过
3. 简历长度控制在一页以内最佳
4. 提醒用户照片文件需要放置在 html 同级目录下
5. 用户个人证件照更换说明：将证件照放在和 html 文件同级目录下，然后根据文件名修改你生成的html中照片的路径。
